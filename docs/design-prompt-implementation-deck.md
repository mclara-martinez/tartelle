## Qué debe comunicar este documento

**Una sola idea central:** El Módulo 2 del roadmap ("Lanzamiento y Equipo", Jun 16 – Jul 25) no es un switch — es una rampa de 5 fases validadas. El equipo nunca deja de operar como siempre; cada semana se agrega una capa nueva solo cuando la anterior está probada.

**Principio guía a destacar visualmente:**
> "Primero duplicamos el trabajo. Luego reemplazamos. Nunca al revés."

## Contenido a incluir

### Sección 1 — Cómo se implementa el Módulo 2

Visualiza las 5 fases como una línea de tiempo horizontal o rampa progresiva. Cada fase tiene:

| Fase | Fechas | Nombre | Quién | Qué cambia |
|------|--------|--------|-------|------------|
| 2a | Jun 16–22 | Pedidos en paralelo | Admin | Doble ingreso Excel + sistema. Nada cambia para el equipo. |
| 2b | Jun 23–29 | Cocina visual | Cocina | Las operarias ven la pantalla de producción. Siguen leyendo el grupo de WhatsApp también. |
| 2c | Jun 30–Jul 6 | Cocina completa | Cocina + admin | Despachos y ventas en sitio en el sistema. WhatsApp pasa a ser respaldo. |
| 2d | Jul 7–13 | Domiciliario + inventario | John + admin | John opera desde su celular. Cierre diario en el sistema. |
| 2e | Jul 14–25 | Flujo completo | Todo el equipo | El sistema es la operación. WhatsApp es emergencia. |

**Elemento visual clave:** Mostrar que en cada fase el proceso anterior NO desaparece — se superpone con el nuevo. Una barra que se va "llenando" o una rampa que sube gradualmente funciona bien.

---

### Sección 2 — Cómo se decide cuándo avanzar

Cada fase tiene una **revisión de validación** antes de pasar a la siguiente. Visualizar esto como un checkpoint o gate entre fases.

Criterios de validación simplificados (usar lenguaje humano, no técnico):

- **2a → 2b:** Los pedidos del sistema coinciden con el Excel en 3 revisiones seguidas
- **2b → 2c:** Lo que dice el sistema en la nevera = lo que hay físicamente
- **2c → 2d:** Despachos y ventas registrados sin errores 3 días seguidos
- **2d → 2e:** John opera solo, el inventario cuadra sin corrección manual
- **2e — meta:** El equipo opera 5 días seguidos sin necesitar WhatsApp para operar

---

### Sección 3 — Nota de pie (pequeña, discreta)

Incluir una nota visual breve que diga:
> Si en cualquier fase algo no funciona, el equipo vuelve al proceso anterior sin perder nada. El sistema se agrega encima — nunca reemplaza algo que aún no está probado.
