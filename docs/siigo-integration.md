## Non-API Siigo Configurations to Set Up Now (No Code)

These are one-time UI configurations in Siigo that directly reduce Andrea's manual work:

### 1. B2B Accounts Receivable — "Gestión de Cobranza"
**What it does:** Automated email/SMS/WhatsApp reminders for overdue invoices.
**Why:** Tartelle has ~45 B2B clients (restaurants/cafes) with 30-day credit terms. Currently Andrea manually follows up via WhatsApp.
**Setup in Siigo UI:** Enable cobranza module → configure payment terms per B2B customer → activate reminder cadence (e.g., due date, +7 days, +15 days).
**Result:** No need to build collection reminders in Tartelle — Siigo handles this.

**Setup:** Create a B2B price list in Siigo with negotiated prices → assign to each B2B customer.
**Result:** When auto-invoicing from Tartelle via API (Phase 5), prices are already correct in Siigo.

## Phase 5: Full API Automation (Code Changes)

> Per `docs/requirements.md`, Siigo automation is Phase 5. Phases 2–4 cover notifications, raw materials, and CRM.

### What Siigo API Enables

| Endpoint | Trigger | Eliminates |
|---|---|---|
| `POST /customers` | New customer created in Tartelle | Manual customer entry in Siigo |
| `POST /invoices` | Order status → `delivered` | End-of-day invoice entry (1 by 1 manual) |
| `POST /credit-notes` | Order cancelled post-delivery | Manual credit note creation |
| `POST /payment-receipts` | `payment_status` = `paid` | Manual payment registration |
| `GET /invoices` | Order drawer / dashboard | Show invoice # + DIAN status in Tartelle |

### Authentication
- **Method:** Username + Access Key → Bearer token (24h TTL)
- **Get credentials:** Siigo UI → Settings → Alliances → Digital Platform Integration Credentials
- **Requires:** Emprendedor plan or higher (~$191,327 COP/month)
- **CRITICAL:** Never in frontend code — use Supabase Edge Function `siigo-sync`

### Architecture
```
Tartelle frontend
    → updateOrderStatus(id, 'delivered')
    → Supabase updates orders table
    → supabase.functions.invoke('siigo-sync', { action: 'create_invoice', order_id })
    → Edge Function: auth → POST /invoices → store siigo_invoice_id back in orders
```

### DB Changes Needed (Phase 5)
```sql
ALTER TABLE orders ADD COLUMN siigo_invoice_id text;
ALTER TABLE orders ADD COLUMN siigo_synced_at timestamptz;
ALTER TABLE customers ADD COLUMN siigo_customer_id text;
```

### Invoice Payload Mapping

| Siigo field | Source |
|---|---|
| `customer.identification` | `customers.cedula` or `customers.nit` |
| `customer.name` | `customers.name` |
| `items[].code` | `products.sku` (PT1, PT2, etc.) |
| `items[].quantity` | `order_items.quantity` |
| `items[].price` | `order_items.unit_price` |
| `items[].discount` | derived from `orders.discount` |
| delivery fee line item | `orders.delivery_fee` as Dom01 SKU |
| `payment_type` | `orders.payment_method` mapped to Siigo type |
| `due_date` | immediate (B2C) or delivery_date + credit days (B2B) |

### Files to Modify (Phase 5)

| File | Change |
|---|---|
| `app/src/lib/types.ts` | Add `siigo_invoice_id`, `siigo_synced_at`, `siigo_customer_id` |
| `app/src/lib/siigo.ts` | New: Siigo API client (auth cache, createInvoice, createCustomer, createCreditNote) |
| `app/src/hooks/useOrders.ts` | After `delivered` → invoke `siigo-sync` |
| `app/src/hooks/useCustomers.ts` | After `createCustomer()` → sync to Siigo |
| `app/src/components/OrderDrawer.tsx` | Show invoice # badge + link when `siigo_invoice_id` set |
| Supabase Edge Function | `siigo-sync` — server-side Siigo calls |
| Supabase Vault | Siigo credentials (never in code) |
