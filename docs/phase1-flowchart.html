3
Production — 6 PM Cutoff & Next-Day Plan
Who: Admin generates, Kitchen executes  |  When: 6 PM daily + next morning  |  Where: Production View + Kitchen Display
Step 3.1 — 6 PM Order Cutoff
Admin 📋 Review confirmed orders for tomorrow
System 🤖 Auto-generate production list
Confirmed orders for tomorrow (exact quantities by product) + Estimated immediate delivery units (configurable per product) = Total to produce per product / flavor / size
Admin 📱 Send production plan via WhatsApp group (manual in Phase 1)
Phase 2: auto-send via WhatsApp Business API at 6 PM
System Plan visible on Kitchen Display for next morning
Kitchen 🌅 First staff arrives 6 AM ➜ 👀 View production list on Kitchen Display
Kitchen 🧁 Produce tarts ➜ ❄️ Refrigerate overnight ➜ ✅ Ready for tomorrow
Kitchen 📦 Register production in system: Inventory "+" button per product
System 📊 Update inventory_finished in real-time ➜ Log to inventory_log (reason: "produccion")

Alert visible on Kitchen Display (red banner) and Dashboard (alert card)
5
Inventory — Continuous Control
Who: Kitchen (during day) + Admin (review)  |  When: All day + evening close  |  Where: Kitchen Display + Inventory View
5.1 — Inventory Adjustments (Kitchen, anytime)
Production (+)
Finished a batch → add units. Logged as "produccion".
Waste / Merma (-)
Product damaged → subtract. Logged as "waste".
Manual Adjustment
Physical count mismatch → correct. Logged as "adjustment".
Direct Sale (-)
Walk-in purchase without formal order → subtract. Logged as "sale".
System Threshold: configurable (default 2 units)
🔴 Kitchen Display: red banner at top 📊 Dashboard: alert card 📦 Inventory View: product highlighted red
Message: product name + remaining quantity + "desactivar en Rappi"
Kitchen 📋 Physical inventory count ➜ Matches system?
Yes Done for the day
No Manual adjustment with explanatory note
Replaces the end-of-day WhatsApp message to inventory group

Status Transition Rules
Only sequential advancement allowed — no skipping states
Status Color Changed By When Can Cancel?
Pendiente Yellow System (auto on create) Order just created Yes
Confirmado Teal Admin Payment verified / credit approved Yes
En Produccion Purple Admin or Kitchen Preparation started Yes
Listo Green Kitchen Packed & ready No
Despachado Blue Kitchen Handed off to delivery / customer No
Entregado Dark Green Admin Delivery confirmed No
Cancelado Red Admin Client cancels / no-show —
  • 🚫 Cannot skip states — must advance sequentially
  • 🚫 Cannot cancel if Listo, Despachado, or Entregado
  • 🔄 On cancel: if inventory was already deducted, system auto-reverses the deduction
